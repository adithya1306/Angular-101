Chapter 6 - HTTP Requests & API Communication

1. Introduction

The HttpClient module in Angular provides a powerful and consistent API for making HTTP requests to servers.

It supports:

  • CRUD operations
  • Interceptors
  • Typed responses
  • Error handling
  • Request cancellation
  • Observables (RxJS)

To use it:

import { HttpClientModule } from '@angular/common/http';

@NgModule({
  imports: [HttpClientModule],
})
export class AppModule {}

2. HttpClient Basics

a. Injecting HttpClient

You can inject HttpClient into any service or component.

import { HttpClient } from '@angular/common/http';
import { Injectable } from '@angular/core';
import { Observable } from 'rxjs';
import { User } from '../models/user.model';

@Injectable({ providedIn: 'root' })
export class UserService {
  private baseUrl = 'https://api.example.com/users';

  constructor(private http: HttpClient) {}

  getUsers(): Observable<User[]> {
    return this.http.get<User[]>(this.baseUrl);
  }

  getUserById(id: number): Observable<User> {
    return this.http.get<User>(`${this.baseUrl}/${id}`);
  }

  createUser(user: User): Observable<User> {
    return this.http.post<User>(this.baseUrl, user);
  }

  updateUser(user: User): Observable<User> {
    return this.http.put<User>(`${this.baseUrl}/${user.id}`, user);
  }

  deleteUser(id: number): Observable<void> {
    return this.http.delete<void>(`${this.baseUrl}/${id}`);
  }
}

4. Subscribing to Observables

this.userService.getUsers().subscribe({
  next: (data) => (this.users = data),
  error: (err) => console.error('Error:', err),
  complete: () => console.log('Request complete')
});

You can also use the async pipe (preferred for cleaner code):

<div *ngFor="let user of userService.getUsers() | async">
  {{ user.name }}
</div>

5. HTTP Options (Headers, Params, etc.)

import { HttpHeaders, HttpParams } from '@angular/common/http';

const headers = new HttpHeaders({
  'Content-Type': 'application/json',
  Authorization: 'Bearer token_here'
});

const params = new HttpParams().set('page', '1').set('limit', '10');

this.http.get('/api/users', { headers, params }).subscribe();

6. Handling Errors

Angular provides the catchError operator from RxJS for elegant error handling.

import { catchError, throwError } from 'rxjs';

getUsers(): Observable<User[]> {
  return this.http.get<User[]>(this.baseUrl).pipe(
    catchError((error) => {
      console.error('Error occurred:', error);
      return throwError(() => new Error('Something went wrong!'));
    })
  );
}

You can also centralize this in interceptors (see below).

7. HTTP Interceptors

Interceptors allow you to modify requests and responses globally, for example:

  • Adding JWT tokens
  • Logging
  • Handling errors globally
  • Showing loading indicators

Example: JWT Auth Interceptor

import { Injectable } from '@angular/core';
import {
  HttpInterceptor,
  HttpRequest,
  HttpHandler,
  HttpEvent
} from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable()
export class AuthInterceptor implements HttpInterceptor {
  intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
    const token = localStorage.getItem('token');

    if (token) {
      const cloned = req.clone({
        setHeaders: { Authorization: `Bearer ${token}` }
      });
      return next.handle(cloned);
    }

    return next.handle(req);
  }
}

Register the Interceptor

import { HTTP_INTERCEPTORS } from '@angular/common/http';

@NgModule({
  providers: [
    { provide: HTTP_INTERCEPTORS, useClass: AuthInterceptor, multi: true }
  ]
})
export class AppModule {}

8. Global Error Interceptor

import { Injectable } from '@angular/core';
import {
  HttpInterceptor,
  HttpRequest,
  HttpHandler,
  HttpEvent,
  HttpErrorResponse
} from '@angular/common/http';
import { Observable, catchError, throwError } from 'rxjs';

@Injectable()
export class ErrorInterceptor implements HttpInterceptor {
  intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
    return next.handle(req).pipe(
      catchError((error: HttpErrorResponse) => {
        let message = '';
        switch (error.status) {
          case 400: message = 'Bad Request'; break;
          case 401: message = 'Unauthorized'; break;
          case 404: message = 'Not Found'; break;
          case 500: message = 'Server Error'; break;
          default: message = 'Unexpected Error'; break;
        }
        console.error('Error:', message);
        return throwError(() => new Error(message));
      })
    );
  }
}

9. Request Cancellation

You can use RxJS takeUntil or AbortController to cancel pending requests when navigating away or destroying a component.

private destroy$ = new Subject<void>();

ngOnInit() {
  this.userService.getUsers()
    .pipe(takeUntil(this.destroy$))
    .subscribe(users => this.users = users);
}

ngOnDestroy() {
  this.destroy$.next();
  this.destroy$.complete();
}

10. Typed API Responses

HttpClient supports generic types, ensuring type safety:

this.http.get<User[]>(this.baseUrl);

Now users will automatically be inferred as User[].

11. Using Environment Files

Store base URLs or API keys in Angular environments:

// environment.ts
export const environment = {
  apiUrl: 'https://api.example.com'
};
this.http.get(`${environment.apiUrl}/users`);

12. Practical Example – CRUD with Error Handling

getUsers(): Observable<User[]> {
  return this.http.get<User[]>(`${environment.apiUrl}/users`).pipe(
    catchError(err => {
      console.error('Error fetching users:', err);
      return throwError(() => err);
    })
  );
}

createUser(user: User): Observable<User> {
  return this.http.post<User>(`${environment.apiUrl}/users`, user);
}

13. Industry Tips

✅ Always use services for API communication (never in components).

✅ Prefer async pipe to avoid manual subscribe/unsubscribe.

✅ Use interceptors for authentication, logging, and global error handling.

✅ Keep base URLs in environment files.

✅ Use RxJS operators (map, tap, catchError, switchMap, etc.) for clean reactive flows.
