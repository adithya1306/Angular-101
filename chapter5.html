Chapter 5 - Services and Dependency Injection

1. What is a Service?

A service in Angular is a singleton class that encapsulates reusable business logic or data.

It‚Äôs mainly used to:

  • Share data between components
  • Handle HTTP requests
  • Implement business logic
  • Manage state or caching
  • Wrap reusable utility functions

üí¨ In short: Services = Logic layer outside of components.

2. Why Use Services?

Components should focus on UI and user interaction, not data handling or business logic.

Example: Instead of fetching data directly in a component:

this.http.get('/api/users');

You create a UserService that handles it:

getUsers() {
  return this.http.get('/api/users');
}

Then the component just calls the service:

this.userService.getUsers().subscribe({
	next: (response => {
		this.users = response;
	}	
);

‚úÖ Easier testing

‚úÖ Better separation of concerns

3. Creating a Service

Using Angular CLI:

ng generate service user
# or shorthand
ng g s user

This generates user.service.ts with:

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class UserService {
  constructor() { }
}

4. The @Injectable() Decorator

@Injectable() marks a class as a service that can be injected by Angular‚Äôs Dependency Injection system.

  • The providedIn: 'root' syntax registers it in the root injector, meaning:
    • A single instance is shared app-wide.
    • No need to manually add it in providers[] array in app.module.ts.

Other options:

@Injectable({
  providedIn: 'any'   // creates one instance per lazy-loaded module
})
@Injectable({
  providedIn: 'platform' // instance shared across multiple Angular apps on same page
})

5. Using a Service in a Component

import { Component, OnInit } from '@angular/core';
import { UserService } from './user.service';

@Component({
  selector: 'app-user',
  templateUrl: './user.component.html'
})
export class UserComponent implements OnInit {

  users: any[] = [];

  // inject service via constructor
  constructor(private userService: UserService) {}

  ngOnInit(): void {
    this.userService.getUsers().subscribe((data) => this.users = data);
  }
}

6. How Dependency Injection Works

Dependency Injection (DI) is a design pattern where an object‚Äôs dependencies are provided externally instead of being created inside it.

Angular uses a hierarchical injector system:

  • Each module, component, or directive can have its own injector.
  • If a dependency isn‚Äôt found in a local injector, Angular looks up the tree to find it (usually at root).

7. Providing Services

Three main ways to register a service:

a) Provided in root (Recommended)

@Injectable({ providedIn: 'root' })

b) Provided in module

@NgModule({
  providers: [UserService]
})

c) Provided in component

@Component({
  providers: [UserService]
})
‚ö†Ô∏è Note: Providing in a component creates a new instance of the service for that component only ‚Äî not shared globally.

8. Example: Shared Service Between Components

user.service.ts

@Injectable({ providedIn: 'root' })
export class UserService {
  private users: string[] = ['Adithya', 'Raj', 'Priya'];

  getUsers() {
    return this.users;
  }

  addUser(name: string) {
    this.users.push(name);
  }
}

component-a.ts

constructor(private userService: UserService) {}
addUser() { this.userService.addUser('Vikram'); }

component-b.ts

constructor(private userService: UserService) {}
ngOnInit() { console.log(this.userService.getUsers()); }

‚úÖ Result: Both components share the same instance of UserService, so data is synced.

10. Using @Optional(), @Self(), @SkipSelf(), @Host() Decorators

These decorators help control how DI resolves dependencies.

DecoratorDescription
@Optional()If service not found, Angular injects null instead of throwing error
@Self()Looks for dependency only in the local injector
@SkipSelf()Skips the current injector, looks in parent
@Host()Looks in current or host component‚Äôs injector

Example:

constructor(@Optional() private logger?: LoggerService) {}

11. Value Providers (Custom Injection Tokens)

When you want to inject values, not services.

import { InjectionToken } from '@angular/core';

export const API_URL = new InjectionToken<string>('API_URL');

@NgModule({
  providers: [{ provide: API_URL, useValue: 'https://api.example.com' }]
})
export class AppModule {}

Then inject:

constructor(@Inject(API_URL) private apiUrl: string) {
  console.log(this.apiUrl);
}

12. Factory Providers

Used when you need dynamic creation logic for dependencies.

providers: [
  {
    provide: LoggerService,
    useFactory: () => environment.production ? new ProdLogger() : new DevLogger()
  }
]

13. Multi Providers

You can provide multiple values for the same token.

export const LOGGER = new InjectionToken<string[]>('LOGGER');

@NgModule({
  providers: [
    { provide: LOGGER, useValue: 'ConsoleLogger', multi: true },
    { provide: LOGGER, useValue: 'FileLogger', multi: true }
  ]
})

Result when injected ‚Üí [ 'ConsoleLogger', 'FileLogger' ]

‚úÖ Key Takeaways

  • Services encapsulate shared business logic.
  • @Injectable() makes a class available for dependency injection.
  • Angular uses a hierarchical injector system.
  • providedIn: 'root' is the modern and efficient way to register services.
  • @Optional(), @Self(), @SkipSelf(), @Host() fine-tune DI resolution.
  • You can inject both custom and built-in services easily.
