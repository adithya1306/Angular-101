üß± A. Structural Directives

They‚Äôre prefixed with * and manipulate the DOM structure.

1. ngIf

Conditionally adds or removes elements from the DOM.

<p *ngIf="isLoggedIn; else guest">Welcome back!</p>
<ng-template #guest>
  <p>Please log in.</p>
</ng-template>

Short form using else:

<div *ngIf="user; else noUser">
  Hello {{ user.name }}
</div>
<ng-template #noUser>No user found!</ng-template>

2. ngFor

Used to loop through a collection.

<ul>
  <li *ngFor="let user of users; index as i">
    {{ i + 1 }}. {{ user.name }}
  </li>
</ul>

With trackBy for performance:

<li *ngFor="let user of users; trackBy: trackByUserId">{{ user.name }}</li>
trackByUserId(index: number, user: any) {
  return user.id;
}

3. ngSwitch

Used for multiple conditional views.

<div [ngSwitch]="role">
  <p *ngSwitchCase="'admin'">Welcome Admin!</p>
  <p *ngSwitchCase="'user'">Welcome User!</p>
  <p *ngSwitchDefault>Guest Access</p>
</div>

The new @if, @for, and @switch template control flow syntax were introduced in Angular 17 as a modern alternative to the older *ngIf, *ngFor, and *ngSwitch directives.

Modern Control Flow Syntax (Angular 17+)

1. Why Angular Introduced @if and @for

Angular‚Äôs new block syntax:

  • Uses JavaScript-like control flow for cleaner templates.
  • Provides better performance and type checking.
  • Eliminates the and <ng-container> boilerplate.

2. @if Block

Old way:

<div *ngIf="user; else guest">
  Welcome, {{ user.name }}!
</div>
<ng-template #guest>
  Please log in.
</ng-template>

New way:

@if (user) {
  <div>Welcome, {{ user.name }}!</div>
} @else {
  <div>Please log in.</div>
}

‚úÖ Benefits:

  • Readable and more natural syntax.
  • No need for <ng-template> or template refs.

3. @for Block

Old way:

<li *ngFor="let item of items; trackBy: trackById">
  {{ item.name }}
</li>

New way:

@for (item of items; track item.id) {
  <li>{{ item.name }}</li>
} @empty {
  <li>No items found.</li>
}

‚úÖ Features:

  • Supports track for efficient rendering (same as trackBy).
  • @empty block for handling empty arrays.
  • Better performance and less syntax clutter.

4. @switch Block

Old way:

<div [ngSwitch]="role">
  <p *ngSwitchCase="'admin'">Admin Access</p>
  <p *ngSwitchCase="'user'">User Access</p>
  <p *ngSwitchDefault>Guest Access</p>
</div>

New way:

@switch (role) {
  @case ('admin') {
    <p>Admin Access</p>
  }
  @case ('user') {
    <p>User Access</p>
  }
  @default {
    <p>Guest Access</p>
  }
}

‚ö†Ô∏è Backward Compatibility

  • The new syntax coexists with ngIf, ngFor, etc.
  • You can safely mix both in the same template.
  • Works automatically from Angular v17+; no configuration needed.

So you can think of it like:

üîÑ @if, @for, and @switch are the modern, block-style replacements for Angular‚Äôs structural directives.

üé® B. Attribute Directives

These modify the appearance or behavior of elements without changing the DOM structure.

1. ngClass

Conditionally apply CSS classes.

<div [ngClass]="{'active': isActive, 'inactive': !isActive}">
  Status: {{ isActive ? 'Active' : 'Inactive' }}
</div>

2. ngStyle

Apply inline styles dynamically.

<p [ngStyle]="{'color': isOnline ? 'green' : 'red'}">
  User is {{ isOnline ? 'Online' : 'Offline' }}
</p>

3. Custom Attribute Directive Example

Create a directive that highlights text on hover:

ng g directive highlight
import { Directive, ElementRef, HostListener, Input } from '@angular/core';

@Directive({
  selector: '[appHighlight]'
})
export class HighlightDirective {
  @Input() appHighlight: string = 'yellow';

  constructor(private el: ElementRef) {}

  @HostListener('mouseenter') onMouseEnter() {
    this.highlight(this.appHighlight);
  }

  @HostListener('mouseleave') onMouseLeave() {
    this.highlight('');
  }

  private highlight(color: string) {
    this.el.nativeElement.style.backgroundColor = color;
  }
}

Usage:

<p appHighlight="lightblue">Hover over me!</p>

4. HostListener & HostBinding

Used within directives to interact with the DOM.

Example:

@HostBinding('style.backgroundColor') bgColor: string = '';

@HostListener('mouseenter') onEnter() {
  this.bgColor = 'lightgreen';
}

@HostListener('mouseleave') onLeave() {
  this.bgColor = '';
}

üßÆ C. Pipes

Pipes are used to transform data directly in the template without changing the component logic.

They‚Äôre like filters in other frameworks.

2. Chaining Pipes

You can apply multiple pipes in sequence.

3. Parameterized Pipes

Some pipes accept arguments.

{{ today | date:'fullDate' }}

4. Custom Pipe Example

ng g pipe truncate
import { Pipe, PipeTransform } from '@angular/core';

@Pipe({
  name: 'truncate'
})
export class TruncatePipe implements PipeTransform {
  transform(value: string, limit: number = 20): string {
    return value.length > limit ? value.substring(0, limit) + '...' : value;
  }
}

Usage:

<p>{{ 'Angular is a great framework' | truncate:10 }}</p>

Output:

Angular is...

5. Pure vs Impure Pipes

  • Pure Pipes: Run only when input changes. (Default)
  • Impure Pipes: Run on every change detection cycle ‚Üí use with caution.
@Pipe({
  name: 'myPipe',
  pure: false
})

  • Custom directives and pipes give Angular immense flexibility.
