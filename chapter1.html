1. What is Angular?

Angular is a TypeScript-based, component-driven front-end framework developed and maintained by Google for building single-page applications (SPAs).

It provides tools for:

  • Two-way data binding
  • Dependency Injection
  • Routing and HTTP client
  • Reactive forms
  • Testing support
  • Performance optimizations (AOT, lazy loading)

üîπ 2.1 Modules (NgModule)

  • A container that groups related components, directives, and services.
  • Each app has a root module (AppModule), and optionally feature modules.
  • Declared with the @NgModule decorator.

Example:

@NgModule({
  declarations: [AppComponent, UserComponent],
  imports: [BrowserModule, FormsModule],
  providers: [UserService],
  bootstrap: [AppComponent]
})
export class AppModule {}

üîπ 2.2 Components

  • The building blocks of Angular UI.
  • Each component has:
    • A TypeScript class (logic)
    • An HTML template (view)
    • Optional CSS styles
    • A metadata decorator @Component

Example:

@Component({
  selector: 'app-user',
  templateUrl: './user.component.html',
  styleUrls: ['./user.component.css']
})
export class UserComponent {
  username = 'Adithya';
}

Template (user.component.html):

<h2>Hello {{ username }}!</h2>

üîπ 2.3 Templates

  • Templates define the view of the component using HTML + Angular syntax.
  • You can use interpolation, directives, and binding syntax.

üîπ 2.4 Metadata

  • Metadata tells Angular how to process a class.
  • Added using decorators:
    • @Component() for components
    • @NgModule() for modules
    • @Injectable() for services

üîπ 2.5 Data Binding (Very Important)

Angular supports 4 types of data binding üëá

1Ô∏è‚É£ Interpolation (One-way binding from TS ‚Üí Template)

Used to display component data in the view.

<p>Welcome {{ username }}</p>

export class AppComponent {
  username = 'Adithya';
}

‚úÖ Updates automatically when username changes.

2Ô∏è‚É£ Property Binding (One-way binding from TS ‚Üí HTML attribute)

Used to bind component properties to HTML element or directive properties.

<img [src]="profilePicUrl" [alt]="username">
export class AppComponent {
  profilePicUrl = 'assets/adithya.png';
  username = 'Adithya';
}

‚úÖ The square brackets [ ] denote property binding.

3Ô∏è‚É£ Event Binding (Template ‚Üí Component)

Used to handle user input or DOM events.

<button (click)="onGreet()">Greet</button>
export class AppComponent {
  onGreet() {
    alert('Hello Adithya!');
  }
}

‚úÖ Parentheses ( ) denote event binding.

4Ô∏è‚É£ Two-Way Binding (TS ‚Üî Template)

Used to synchronize data between the component and view.

You need to import FormsModule for this.

<input [(ngModel)]="username" placeholder="Enter name">
<p>Hello {{ username }}!</p>
export class AppComponent {
  username = '';
}

‚úÖ [( )] = Banana in a box üçå ‚Äî combines property + event binding.

üîπ 2.6 Directives

Directives modify DOM behavior or structure.

  • Structural Directives ‚Äî Change the DOM layout.
    • ngIf, ngFor, ngSwitch
  • Attribute Directives ‚Äî Change element appearance or behavior.
    • ngStyle, ngClass

Example:

<p *ngIf="isLoggedIn">Welcome {{ username }}</p>
<li *ngFor="let user of users">{{ user }}</li>

üîπ 2.7 Services

  • Used for business logic, data sharing, and API calls.
  • Decorated with @Injectable() and injected using Angular‚Äôs Dependency Injection (DI).

Example:

@Injectable({ providedIn: 'root' })
export class UserService {
  getUsers() {
    return ['Adithya', 'Ravi', 'Priya'];
  }
}

Injected into a component:

export class UserComponent {
  users: string[];
  constructor(private userService: UserService) {
    this.users = userService.getUsers();
  }
}

üîπ 2.8 Dependency Injection (DI)

  • Angular‚Äôs built-in design pattern to inject dependencies instead of manually creating them.

Without DI:

const service = new UserService(); // tight coupling

With DI:

constructor(private userService: UserService) {}

‚úÖ Angular creates and provides the instance automatically.

This uses the component schematic internally to create .ts, .html, .css, and .spec.ts files automatically.
