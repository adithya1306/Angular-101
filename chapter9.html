Chapter 9 - Forms 101

üîπ 9.1 Introduction

Forms are core to any application that involves user input, like login pages, registration, feedback, or data entry.

Angular provides two different approaches for building forms:

ApproachBest forKey Feature
Template-driven formsSimple formsDefined mostly in HTML, uses ngModel
Reactive formsComplex / dynamic formsDefined in TypeScript using FormGroup, FormControl

üîπ 9.2 Template-driven Forms

Template-driven forms are declarative ‚Äî most of the form logic lives inside the template.

üß± Setup

First, import FormsModule in your feature module (or AppModule):

import { FormsModule } from '@angular/forms';

@NgModule({
  imports: [FormsModule],
})
export class AppModule {}

‚úçÔ∏è Example: Login Form

HTML:

<form #loginForm="ngForm" (ngSubmit)="onSubmit(loginForm)">
  <input
    type="text"
    name="username"
    [(ngModel)]="user.username"
    required
    placeholder="Username"
  />

  <input
    type="password"
    name="password"
    [(ngModel)]="user.password"
    required
    placeholder="Password"
  />

  <button type="submit" [disabled]="!loginForm.valid">Login</button>
</form>

Component:

export class LoginComponent {
  user = { username: '', password: '' };

  onSubmit(form: any) {
    console.log('Form Values:', form.value);
  }
}

‚úÖ Key Directives

DirectiveDescription
ngModelTwo-way data binding between input and component
ngFormAutomatically tracks the form‚Äôs state (valid, dirty, touched)
required, minlength, patternBuilt-in validators
[disabled]="!form.valid"Reactive template control

‚ö†Ô∏è Limitations

  • Hard to scale with complex validation logic
  • Difficult to test
  • Validation logic spread between template and component

Hence ‚Äî Reactive Forms are preferred for large projects.

üîπ 9.3 Reactive Forms

Reactive Forms are programmatic ‚Äî you define the form structure in the component using TypeScript.

üß± Setup

Import ReactiveFormsModule:

import { ReactiveFormsModule } from '@angular/forms';

@NgModule({
  imports: [ReactiveFormsModule],
})
export class AppModule {}

‚úçÔ∏è Example: Registration Form

Component:

import { Component } from '@angular/core';
import { FormBuilder, FormGroup, Validators } from '@angular/forms';

@Component({
  selector: 'app-register',
  templateUrl: './register.component.html'
})
export class RegisterComponent {
  registerForm: FormGroup;

  constructor(private fb: FormBuilder) {
    this.registerForm = this.fb.group({
      name: ['', [Validators.required, Validators.minLength(3)]],
      email: ['', [Validators.required, Validators.email]],
      password: ['', [Validators.required, Validators.minLength(6)]],
      confirmPassword: ['']
    });
  }

  onSubmit() {
    console.log(this.registerForm.value);
  }
}

Template:

<form [formGroup]="registerForm" (ngSubmit)="onSubmit()">
  <input formControlName="name" placeholder="Name" />
  <div *ngIf="registerForm.get('name')?.invalid && registerForm.get('name')?.touched">
    Name is required (min 3 characters)
  </div>

  <input formControlName="email" placeholder="Email" />
  <div *ngIf="registerForm.get('email')?.hasError('email')">
    Invalid email address
  </div>

  <input type="password" formControlName="password" placeholder="Password" />
  <button type="submit" [disabled]="registerForm.invalid">Register</button>
</form>

‚úÖ Key Classes

ClassPurpose
FormGroupTracks a group of controls (like an object)
FormControlTracks the value and validation status of a single control
FormArrayManages an array of FormControls (dynamic lists)
FormBuilderShorthand for creating form groups and controls

üîπ 9.4 Accessing Form Controls

const emailControl = this.registerForm.get('email');
console.log(emailControl?.value);
console.log(emailControl?.valid);

üîπ 9.5 Validation (Reactive)

Angular provides built-in validators and allows custom validators.

üß± Built-in Validators

Validators.required
Validators.minLength(3)
Validators.maxLength(10)
Validators.email
Validators.pattern('[A-Za-z]+')

üß© Custom Validator Example

import { AbstractControl, ValidationErrors } from '@angular/forms';

export function noSpaceValidator(control: AbstractControl): ValidationErrors | null {
  return control.value?.includes(' ') ? { noSpace: true } : null;
}

Usage:

this.form = this.fb.group({
  username: ['', [Validators.required, noSpaceValidator]]
});

üîπ 9.6 Dynamic Forms with FormArray

When you need to add/remove fields dynamically (e.g., multiple phone numbers):

phones: FormArray;

constructor(private fb: FormBuilder) {
  this.phones = this.fb.array([this.fb.control('')]);
}

addPhone() {
  this.phones.push(this.fb.control(''));
}

Template:

<div formArrayName="phones">
  <div *ngFor="let phone of phones.controls; let i = index">
    <input [formControlName]="i" placeholder="Phone number {{ i+1 }}" />
  </div>
</div>
<button (click)="addPhone()">Add Another</button>

üîπ 9.7 Observing Value Changes

Reactive forms expose Observables to listen to input changes.

this.registerForm.get('email')?.valueChanges.subscribe(value => {
  console.log('Email changed:', value);
});

You can use this to:

  • Dynamically show suggestions
  • Trigger conditional validations
  • Integrate with APIs

üîπ 9.8 Resetting and Patching Values

this.registerForm.reset(); // clears all
this.registerForm.patchValue({ name: 'Adithya' }); // update specific field

üîπ 9.9 Async Validators

Used for server-side validation (e.g., checking if a username is already taken):

username: ['', null, this.usernameTakenValidator.bind(this)]

usernameTakenValidator(control: AbstractControl) {
  return this.http.get(`/api/check-username/${control.value}`).pipe(
    map((res: any) => res.exists ? { usernameTaken: true } : null)
  );
}

üí° Best Practices

  1. Use Reactive Forms for scalable, testable applications.
  1. Create custom form components for reusability.
  1. Always provide form validation feedback to users.
  1. Use FormBuilder instead of manually creating FormControls.
  1. Group related fields logically using FormGroup.
