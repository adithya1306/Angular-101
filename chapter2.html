Chapter 2 - TypeScript in Angular

1. Why TypeScript for Angular

Angular is written entirely in TypeScript ‚Äî a superset of JavaScript that adds:

  • Static typing
  • Interfaces and generics
  • Classes and access modifiers
  • Decorators
  • Tooling (autocompletion, type checking)

üëâ It compiles down to JavaScript for browser execution, but adds powerful features for large-scale, maintainable applications.

2. Core TypeScript Concepts

2.1 Variables and Type Annotations

You can declare variables with explicit types for safety and clarity.

let username: string = 'Adithya';
let age: number = 22;
let isAdmin: boolean = true;
let skills: string[] = ['Angular', 'Spring Boot'];

Type inference:

let city = 'Chennai'; // inferred as string automatically

2.2 Functions and Return Types

You can define parameter and return types.

function greet(name: string): string {
  return `Hello, ${name}!`;
}

const result = greet('Adithya');

Functions can also have optional and default parameters:

function add(a: number, b: number = 5): number {
  return a + b;
}

Arrow functions (commonly used in Angular components):

const double = (x: number): number => x * 2;

2.3 Interfaces

Interfaces define contracts for object structure ‚Äî extremely useful for defining API models or data types in Angular services.

interface User {
  id: number;
  name: string;
  email: string;
}

const user: User = {
  id: 1,
  name: 'Adithya',
  email: 'adithya@example.com'
};

Used in Angular components:

export class UserComponent {
  user: User = { id: 1, name: 'Adithya', email: 'adithya@example.com' };
}

‚úÖ Helps with type safety and code completion in editors.

2.4 Classes and Access Modifiers

Angular relies heavily on classes ‚Äî every component, service, and module is a class.

Example:

class User {
  constructor(
    public name: string,
    private age: number
  ) {}

  getUserInfo(): string {
    return `${this.name} is ${this.age} years old.`;
  }
}

const user = new User('Adithya', 22);
console.log(user.getUserInfo());

2.5 Inheritance & Abstract Classes

Angular leverages inheritance for base components, reusable logic, etc.

abstract class BaseComponent {
  abstract loadData(): void;
  log(message: string) {
    console.log('Log:', message);
  }
}

class DashboardComponent extends BaseComponent {
  loadData() {
    this.log('Dashboard data loaded');
  }
}

‚úÖ Use case in Angular: Common logic like logging, authentication checks, etc., can be abstracted into a base class for multiple components.

2.6 Generics

Generics allow reusable functions or classes that work with multiple data types.

function identity<T>(value: T): T {
  return value;
}

let num = identity<number>(100);
let name = identity<string>('Adithya');

Example in Angular (common for reusable services):

getData<T>(url: string): Observable<T> {
  return this.http.get<T>(url);
}

‚úÖ Used with HTTP calls, RxJS observables, and data models for strong typing.

2.7 Enums

Enums are great for defining a set of named constants ‚Äî often used in Angular for roles, statuses, or action types.

enum UserRole {
  ADMIN = 'ADMIN',
  USER = 'USER',
  MENTOR = 'MENTOR'
}

export class AuthService {
  currentRole: UserRole = UserRole.ADMIN;
}

Used in templates:

<p *ngIf="authService.currentRole === 'ADMIN'">Welcome Admin!</p>

2.8 Modules and Imports/Exports

Each Angular file is a TypeScript module.

Example:

// user.model.ts
export interface User {
  id: number;
  name: string;
}
// user.service.ts
import { User } from './user.model';

‚úÖ Keeps code modular, reusable, and easy to maintain.

2.9 Handling Null, Undefined, and Strict Types in Angular

TypeScript uses strict null checks to prevent runtime errors like:

‚ÄúCannot read properties of undefined.‚Äù

This feature forces you to explicitly declare and handle variables that can be null or undefined.

Let‚Äôs go through the important cases you‚Äôll encounter in Angular codebases üëá

1Ô∏è‚É£ Union Types with null or undefined

Used when a variable can have a real value or be empty.

id: number | null = null;

‚úÖ Explanation:

  • The variable id can either hold a number or be null.
  • Prevents accidental use before assignment.

Example use case in Angular:

export class UserComponent {
  userId: number | null = null;

  selectUser(id: number) {
    this.userId = id;
  }
}

2Ô∏è‚É£ The ! (Definite Assignment Assertion)

üîπ Usage:

username!: string;

‚úÖ Explanation:

  • You‚Äôre telling TypeScript: ‚ÄúTrust me, this variable will be assigned before use.‚Äù
  • Prevents the ‚ÄúVariable has no initializer‚Äù compilation error when using strictPropertyInitialization.

Example:

export class ProfileComponent implements OnInit {
  user!: string; // will be initialized later

  ngOnInit() {
    this.user = 'Adithya'; // assigned before first use
  }
}

‚ö†Ô∏è Caution: Only use ! when you are 100% sure the variable will be initialized before use.

Otherwise, it defeats TypeScript‚Äôs safety.

3Ô∏è‚É£ The ? (Optional Property or Method Parameter)

The ? makes a property or parameter optional ‚Äî it may be undefined.

In Interfaces:

interface User {
  id: number;
  email?: string; // optional property
}
const user1: User = { id: 1 };              // valid
const user2: User = { id: 2, email: 'a@b' }; // also valid

In Functions:

function greet(name?: string) {
  console.log(`Hello ${name ?? 'Guest'}`);
}

‚úÖ The ?? operator here gives a fallback value if name is undefined or null.

4Ô∏è‚É£ The Non-Null Assertion Operator (variable!)

Used when you‚Äôre certain that a variable is not null or undefined ‚Äî even though TypeScript thinks it might be.

console.log(user!.email);

‚úÖ Example in Angular Template context:

export class UserComponent {
  user: User | null = null;

  ngOnInit() {
    this.user = { id: 1, name: 'Adithya', email: 'a@b.com' };
  }

  getUserName() {
    console.log(this.user!.name); // Assert: user is not null here
  }
}

‚ö†Ô∏è Again, use carefully ‚Äî if user is null, this will cause a runtime error.

5Ô∏è‚É£ Optional Chaining (?.)

A safer alternative to variable!.

It stops evaluation if a property in the chain is null or undefined.

console.log(this.user?.email);

‚úÖ If this.user is null or undefined, it simply returns undefined instead of throwing an error.

Example in Angular template:

<p>{{ user?.name }}</p>

This will not crash the view even if user is null.

6Ô∏è‚É£ Nullish Coalescing Operator (??)

Provides a default value only when the left-hand side is null or undefined.

const username = user.name ?? 'Guest';

‚úÖ Example:

this.userName = this.user?.name ?? 'Unknown User';

7Ô∏è‚É£ Combined Example in Angular

Let‚Äôs see all of these together üëá

export interface User {
  id: number;
  name?: string;
  email?: string | null;
}

export class ProfileComponent implements OnInit {
  user!: User; // Definite assignment assertion

  ngOnInit() {
    this.user = { id: 1, email: null };
  }

  get displayName(): string {
    // Optional chaining + nullish coalescing
    return this.user.name ?? 'Guest';
  }

  logEmail() {
    // Non-null assertion
    console.log(this.user!.email ?? 'No email found');
  }
}

‚úÖ Safe, flexible, and clean ‚Äî exactly how industry-standard Angular code should look.

3. Decorators (Core of Angular)

Angular uses decorators (TypeScript feature) to attach metadata to classes, methods, or properties.

They are what make components, services, and modules Angular-aware.

3.1 What is a Decorator?

A decorator is a special TypeScript function prefixed with @ that modifies or annotates a class.

3.2 Common Angular Decorators

DecoratorPurpose
@Component()Defines a UI component
@Directive()Defines a custom directive
@Pipe()Defines a reusable pipe
@Injectable()Marks a class as available for DI
@NgModule()Defines a module
@Input() / @Output()Used for component communication

Example: Component Decorator

@Component({
  selector: 'app-user',
  template: `<h2>{{ name }}</h2>`,
  styles: [`h2 { color: blue; }`]
})
export class UserComponent {
  name = 'Adithya';
}

Here, the @Component decorator tells Angular:

  • How to instantiate the class
  • Where to render the component (selector)
  • Which template and styles to use

Example: Injectable Decorator

@Injectable({
  providedIn: 'root'
})
export class UserService {
  getUser() {
    return 'Adithya';
  }
}
‚úÖ Makes the service available for Dependency Injection throughout the app.

Example: Input and Output Decorators

Used for parent-child component communication.

// child.component.ts
@Input() username!: string;
@Output() greet = new EventEmitter<string>();

sendGreeting() {
  this.greet.emit(`Hello ${this.username}`);
}
<!-- parent.component.html -->
<app-child [username]="name" (greet)="onGreet($event)"></app-child>

4. Async Programming in TypeScript

Angular apps are event-driven ‚Äî async code is essential for handling API calls, timers, and user actions.

4.1 Observables (RxJS)

Used for streams of data ‚Äî can emit multiple values over time.

getData(): Observable<number> {
  return interval(1000); // emits values every 1 second
}
this.getData().subscribe(value => console.log(value));

‚úÖ Angular prefers Observables (not Promises) because they support:

  • Continuous data streams
  • Cancellation
  • Operators for transformation and error handling

5. Putting It All Together

Let‚Äôs see how TypeScript concepts fit in an Angular service:

@Injectable({ providedIn: 'root' })
export class UserService {
  private users: User[] = [
    { id: 1, name: 'Adithya', email: 'adithya@example.com' }
  ];

  getUsers(): Observable<User[]> {
    return of(this.users); // RxJS Observable
  }
}

And how a component consumes it:

@Component({
  selector: 'app-user-list',
  template: `<li *ngFor="let user of users">{{ user.name }}</li>`
})
export class UserListComponent implements OnInit {
  users: User[] = [];

  constructor(private userService: UserService) {}

  ngOnInit() {
    this.userService.getUsers().subscribe(data => this.users = data);
  }
}

‚úÖ In summary:

  • TypeScript brings types, OOP, and decorators to Angular.
  • Core Angular concepts (components, services, DI) rely on these features.
  • Mastering TypeScript makes debugging, refactoring, and scaling easier.
