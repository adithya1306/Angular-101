Chapter 8 - Routing, Lazy Loading & Route Guards

1. Introduction to Angular Routing

Routing in Angular allows navigation between different views or components without reloading the entire page — a Single Page Application (SPA) pattern.

The Angular Router:

  • Maps URLs to components
  • Handles navigation
  • Supports lazy loading
  • Protects routes (Auth Guards)
  • Passes route parameters and query params

2. Setting Up Routing

When creating a project:

ng new my-app --routing

This automatically creates app-routing.module.ts.

If not created, you can add routing manually.

3. Basic Routing Example

app-routing.module.ts

import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { HomeComponent } from './home/home.component';
import { AboutComponent } from './about/about.component';

const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'about', component: AboutComponent },
  { path: '**', redirectTo: '' } // wildcard route for 404s
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule {}

app.component.html

<nav>
  <a routerLink="/">Home</a>
  <a routerLink="/about">About</a>
</nav>

<router-outlet></router-outlet>

✅ routerLink — Angular’s version of anchor tag

✅ <router-outlet> — Placeholder where routed components appear

4. Route Parameters

Use :id to pass parameters.

app-routing.module.ts

{ path: 'users/:id', component: UserDetailComponent }

UserDetailComponent.ts

import { ActivatedRoute } from '@angular/router';

constructor(private route: ActivatedRoute) {}

ngOnInit() {
  const id = this.route.snapshot.paramMap.get('id');
  console.log('User ID:', id);
}

Dynamic Link Example

<a [routerLink]="['/users', user.id]">View</a>

5. Query Parameters

Example:

<a [routerLink]="['/products']" [queryParams]="{ sort: 'price', order: 'asc' }">
  Sort Products
</a>

Reading Query Params:

this.route.queryParamMap.subscribe(params => {
  console.log(params.get('sort'));
});

7. Programmatic Navigation

You can navigate from code using the Router service.

import { Router } from '@angular/router';

constructor(private router: Router) {}

goToProfile(id: number) {
  this.router.navigate(['/profile', id]);
}

goWithQuery() {
  this.router.navigate(['/products'], { queryParams: { sort: 'price' } });
}

8. Lazy Loading (Performance Optimization)

Lazy loading helps load modules only when needed, reducing initial bundle size and improving app startup time.

Step 1: Create a Feature Module

ng g module admin --route admin --module app.module

This automatically sets up lazy loading in routes.

Step 2: app-routing.module.ts

const routes: Routes = [
  { path: '', component: HomeComponent },
  {
    path: 'admin',
    loadChildren: () =>
      import('./admin/admin.module').then(m => m.AdminModule)
  }
];

Step 3: admin-routing.module.ts

const routes: Routes = [
  { path: '', component: AdminDashboardComponent },
  { path: 'users', component: AdminUsersComponent }
];

✅ The admin module and its components load only when /admin is accessed.

9. Preloading Strategies

You can preload lazy-loaded modules after the app starts, to balance speed and performance.

imports: [RouterModule.forRoot(routes, { preloadingStrategy: PreloadAllModules })]

This preloads all lazy modules in the background.

10. Route Guards

Guards protect routes or control navigation.

They return:

  • true → allow navigation
  • false → block navigation
  • UrlTree → redirect

Types:

Guard TypeInterfacePurpose
CanActivateCanActivateControl access to route
CanActivateChildCanActivateChildProtect child routes
CanDeactivateCanDeactivate<T>Warn before leaving a route
ResolveResolve<T>Fetch data before route activates
CanLoadCanLoadPrevent loading lazy modules
CanMatchCanMatchConditionally match routes (Angular 15+)

11. Example: Auth Guard (CanActivate)

auth.guard.ts

import { Injectable } from '@angular/core';
import { CanActivate, Router } from '@angular/router';

@Injectable({ providedIn: 'root' })
export class AuthGuard implements CanActivate {
  constructor(private router: Router) {}

  canActivate(): boolean {
    const token = localStorage.getItem('token');
    if (!token) {
      this.router.navigate(['/login']);
      return false;
    }
    return true;
  }
}

app-routing.module.ts

{ path: 'dashboard', component: DashboardComponent, canActivate: [AuthGuard] }

12. Example: Deactivation Guard (CanDeactivate)

Prompt users before leaving a page with unsaved changes.

export interface CanComponentDeactivate {
  canDeactivate: () => boolean | Observable<boolean>;
}

@Injectable({ providedIn: 'root' })
export class CanDeactivateGuard implements CanDeactivate<CanComponentDeactivate> {
  canDeactivate(component: CanComponentDeactivate) {
    return component.canDeactivate ? component.canDeactivate() : true;
  }
}

Usage in Component:

canDeactivate(): boolean {
  return confirm('Do you really want to leave? Unsaved changes will be lost.');
}

In Routes:

{ path: 'edit', component: EditComponent, canDeactivate: [CanDeactivateGuard] }

13. Example: Resolver (Pre-fetch Data Before Loading)

user.resolver.ts

@Injectable({ providedIn: 'root' })
export class UserResolver implements Resolve<User> {
  constructor(private userService: UserService) {}

  resolve(route: ActivatedRouteSnapshot): Observable<User> {
    const id = route.paramMap.get('id');
    return this.userService.getUserById(Number(id));
  }
}

app-routing.module.ts

{ path: 'users/:id', component: UserComponent, resolve: { user: UserResolver } }

In Component

ngOnInit() {
  this.route.data.subscribe(({ user }) => {
    this.user = user;
  });
}

14. CanLoad and CanMatch (for Lazy Modules)

Used to restrict loading of lazy modules.

export class AdminGuard implements CanLoad {
  canLoad(): boolean {
    return localStorage.getItem('role') === 'admin';
  }
}

{ path: 'admin', loadChildren: () => import('./admin/admin.module').then(m => m.AdminModule), canLoad: [AdminGuard] }

CanMatch (Angular 15+) offers more flexibility:

export class RoleGuard implements CanMatch {
  canMatch(): boolean {
    return localStorage.getItem('role') === 'manager';
  }
}

15. Redirects and Wildcards

{ path: '', redirectTo: '/home', pathMatch: 'full' },
{ path: '**', component: PageNotFoundComponent }
  • pathMatch: 'full' ensures exact path match for redirect.
  • * catches undefined routes.

16. Industry Best Practices

✅ Keep routing definitions in separate *-routing.module.ts files.

✅ Use lazy loading for large feature modules.

✅ Protect private routes with Auth Guards.

✅ Use Resolvers for pre-fetching data.

✅ Apply PreloadAllModules for hybrid performance.

✅ Handle 404 with wildcard routes.

✅ Store tokens in localStorage or sessionStorage (never hardcoded).
