Chapter 7 - RxJS 101

1. What is RxJS?

RxJS (Reactive Extensions for JavaScript) is a library for reactive programming using observables.

It allows you to handle asynchronous data streams like API calls, user input, timers, or websockets — all in a consistent way.

Think of RxJS as a way to handle data that changes over time.

2. Core Concepts

ObserverConsumes data emitted by Observable.subscribe(next, error, complete)

3. Observable Creation

From Static Data:

import { of } from 'rxjs';

const numbers$ = of(1, 2, 3);
numbers$.subscribe(val => console.log(val));

From Events:

import { fromEvent } from 'rxjs';

const clicks$ = fromEvent(document, 'click');
clicks$.subscribe(() => console.log('Clicked!'));

From Promises:

import { from } from 'rxjs';

const dataPromise = fetch('/api/users').then(res => res.json());
const data$ = from(dataPromise);
data$.subscribe(console.log);

4. Common RxJS Operators (Industry-Used)

Operators are functions that let you transform, filter, or combine Observables.

5. Transformation Operators

map()

Transforms each emitted value.

of(1, 2, 3).pipe(map(x => x * 2)).subscribe(console.log);
// Output: 2, 4, 6

switchMap()

Cancels the previous observable when a new value arrives (common in HTTP calls & search boxes).

fromEvent(inputEl, 'keyup').pipe(
  debounceTime(300),
  switchMap((event: any) => this.http.get(`/api?q=${event.target.value}`))
);

mergeMap()

Runs multiple observables in parallel.

from(users).pipe(
  mergeMap(user => this.http.get(`/api/profile/${user.id}`))
);

concatMap()

Runs observables sequentially (one after another).

from(tasks).pipe(
  concatMap(task => this.http.post('/api/tasks', task))
);

exhaustMap()

Ignores new values while a previous one is still running.

fromEvent(loginBtn, 'click').pipe(
  exhaustMap(() => this.http.post('/api/login', credentials))
);

6. Filtering Operators

filter()

Filters emitted values.

of(1, 2, 3, 4, 5).pipe(
  filter(x => x % 2 === 0)
).subscribe(console.log);
// Output: 2, 4

debounceTime()

Waits for a pause in emission — perfect for search boxes.

fromEvent(input, 'keyup').pipe(
  debounceTime(300),
  map((e: any) => e.target.value)
);

takeUntil()

Stops emission when another observable emits — used for unsubscribing on destroy.

this.data$.pipe(takeUntil(this.destroy$)).subscribe();

7. Subjects and BehaviorSubjects

Subjects act as both Observable and Observer — they can emit and subscribe.

a. Subject

Doesn’t hold any previous value.

import { Subject } from 'rxjs';

const subject = new Subject<number>();

subject.subscribe(val => console.log('A:', val));
subject.next(1); // A: 1
subject.next(2);

subject.subscribe(val => console.log('B:', val));
subject.next(3); // A: 3, B: 3

b. BehaviorSubject

Holds the latest value and emits it to new subscribers.

import { BehaviorSubject } from 'rxjs';

const user$ = new BehaviorSubject<string>('Guest');

user$.subscribe(console.log); // Guest
user$.next('Adithya'); // Guest -> Adithya

✅ Commonly used for state management or sharing data between components.

c. ReplaySubject

Replays last N values to new subscribers.

const replay$ = new ReplaySubject(2);
replay$.next(1);
replay$.next(2);
replay$.next(3);

replay$.subscribe(console.log); // 2, 3

8. Async Pipe (Best Practice)

Instead of manually subscribing and unsubscribing, use the async pipe in templates:

user$ = this.userService.getUser();
<div *ngIf="user$ | async as user">
  {{ user.name }}
</div>

✅ Automatically subscribes/unsubscribes

✅ Keeps template clean

✅ Prevents memory leaks

9. Combining Observables

combineLatest()

Waits for each observable to emit once, then emits combined results.

combineLatest([this.user$, this.settings$]).subscribe(([user, settings]) => {
  console.log(user.name, settings.theme);
});

forkJoin()

Runs multiple observables in parallel and emits once all complete.

import { forkJoin } from 'rxjs';

forkJoin({
  user: this.http.get('/api/user'),
  orders: this.http.get('/api/orders')
}).subscribe({
  next: (data) => {
    console.log(data.user);   // Response from /api/user
    console.log(data.orders); // Response from /api/orders
  },
  error: (err) => console.error('Error:', err),
  complete: () => console.log('All requests completed!')
});

O/P

{
  "user": { "id": 1, "name": "Adithya" },
  "orders": [{ "id": 101, "item": "Laptop" }]
}

10. Error Handling with RxJS

Use catchError, retry, and retryWhen.

this.http.get('/api/users').pipe(
  retry(2),
  catchError(error => {
    console.error('API failed:', error);
    return throwError(() => new Error('Server error'));
  })
);

11. Common Async Patterns in Angular

Chaining API Calls

this.http.get('/api/user').pipe(
  switchMap((user: any) => this.http.get(`/api/orders/${user.id}`))
);

Handling Loading States

isLoading = new BehaviorSubject<boolean>(false);

loadUsers() {
  this.isLoading.next(true);
  this.http.get('/api/users').pipe(
    finalize(() => this.isLoading.next(false))
  ).subscribe();
}

State Management with BehaviorSubject

private userSource = new BehaviorSubject<User | null>(null);
user$ = this.userSource.asObservable();

setUser(user: User) {
  this.userSource.next(user);
}

12. Key Industry Tips

✅ Always unsubscribe — use takeUntil or async pipe.

✅ Use switchMap for dependent HTTP calls.

✅ Prefer BehaviorSubject for shared state.

✅ Avoid manual subscribe() inside services — return Observables instead.

✅ Use operators like tap() for debugging.

✅ Think in terms of data streams, not static variables.

13. Mini Project Example

Search-as-you-type API Example:

fromEvent(this.searchInput.nativeElement, 'keyup').pipe(
  debounceTime(400),
  map((e: any) => e.target.value),
  distinctUntilChanged(),
  switchMap(query => this.http.get(`/api/search?q=${query}`))
).subscribe(results => this.results = results);
