Chapter 3 - Components 101

1. What is a Component?

A Component is the basic building block of an Angular application.

It controls:

  • A specific view (HTML + CSS)
  • The logic behind that view (TypeScript)
  • Data binding and interactions with other components

Each component in Angular is a TypeScript class decorated with @Component().

2. Anatomy of a Component

Example:

import { Component } from '@angular/core';

@Component({
  selector: 'app-user',
  templateUrl: './user.component.html',
  styleUrls: ['./user.component.css']
})
export class UserComponent {
  userName: string = 'Adithya';
  age: number = 22;

  greetUser() {
    return `Hello ${this.userName}!`;
  }
}
  • selector: Defines the HTML tag used to include this component (<app-user>).
  • templateUrl: Path to the component’s HTML file.
  • styleUrls: Optional; CSS/SCSS files for styling.
  • class: Contains properties and methods (component logic).

3. Creating a Component

Use Angular CLI:

ng generate component component-name

or shorthand:

ng g c component-name

This creates:

component-name/
 ├── component-name.component.ts
 ├── component-name.component.html
 ├── component-name.component.css
 └── component-name.component.spec.ts

4. Component Lifecycle Hooks

Angular components go through a series of lifecycle stages. You can hook into these using special methods.

HookPurpose
ngOnInit()Called once after component initialization — used for data fetching or setup.
ngOnChanges()Called when an input property changes.
ngDoCheck()Custom change detection logic.
ngAfterContentInit() / ngAfterContentChecked()Triggered after projecting content into the component.
ngAfterViewInit() / ngAfterViewChecked()Triggered after component’s view (and child views) are initialized.
ngOnDestroy()Cleanup logic — unsubscribe observables, remove listeners.

Example:

ngOnInit() {
  console.log('Component initialized!');
}

ngOnDestroy() {
  console.log('Component destroyed!');
}

5. Templates

Templates define how a component’s view looks.

You can use:

  • External template: templateUrl: './component.html'
  • Inline template:
    template: `<h2>Hello {{ userName }}</h2>`

6. Template Expressions & Statements

Angular templates support binding expressions inside HTML.

Example:

<h2>{{ userName.toUpperCase() }}</h2>
<p>{{ age > 18 ? 'Adult' : 'Minor' }}</p>
<button (click)="greetUser()">Greet</button>

Rules:

  • No assignments inside expressions.
  • No new variables (only existing component members).
  • Must be pure (no side effects).

7. Property Binding

Used to bind DOM properties to component variables.

<img [src]="imageUrl" [alt]="userName" />

Equivalent of:

imageUrl = 'assets/profile.png';

8. Attribute Binding

Used when you want to bind HTML attributes (not DOM properties).

<button [attr.aria-label]="userName">Click</button>

9. Class Binding

Dynamically add or remove classes.

<div [class.active]="isActive">Active User</div>

Or multiple:

<div [ngClass]="{'highlight': isActive, 'dim': !isActive}"></div>

10. Style Binding

Bind inline styles dynamically.

<p [style.color]="isActive ? 'green' : 'red'">Status</p>

Or multiple:

<p [ngStyle]="{'font-size': fontSize + 'px', 'color': fontColor}">Dynamic Text</p>

11. Event Binding

Bind component methods to DOM events.

<button (click)="onClick()">Click me</button>
onClick() {
  alert('Button clicked!');
}

Event with arguments:

<input (input)="onInput($event)" />
onInput(event: any) {
  console.log(event.target.value);
}

12. Two-Way Binding

For data synchronization between component and template using [(ngModel)].

<input [(ngModel)]="userName" />
<p>Hello {{ userName }}</p>
Remember: You must import FormsModule in your module for ngModel to work.

13. Template Reference Variables

Used to access DOM elements or components from the template.

<input #userInput type="text" />
<button (click)="showValue(userInput.value)">Show</button>
showValue(value: string) {
  console.log(value);
}

14. @Input() and @Output() Decorators

@Input(): Pass data from parent → child

// child.component.ts
@Input() userName!: string;
<!-- parent.component.html -->
<app-child [userName]="parentName"></app-child>

@Output(): Emit data from child → parent

// child.component.ts
@Output() userClicked = new EventEmitter<string>();

onClick() {
  this.userClicked.emit(this.userName);
}
<!-- parent.component.html -->
<app-child (userClicked)="handleUserClick($event)"></app-child>

15. ViewChild and ViewChildren

Used to access child DOM elements or components in TypeScript.

@ViewChild('userInput') inputElement!: ElementRef;

ngAfterViewInit() {
  this.inputElement.nativeElement.focus();
}

16. Content Projection

Allows parent components to inject content into child templates.

<!-- parent -->
<app-card>
  <p>This text is projected into the card!</p>
</app-card>
<!-- app-card.component.html -->
<div class="card">
  <ng-content></ng-content>
</div>

17. Component Communication Summary

Communication TypeDecorator/FeatureDirection
Parent → Child@Input()Downward
Child → Parent@Output() + EventEmitterUpward
Any → AnyService with ObservablesCross-component

✅ Key Takeaways

  • Components are reusable, self-contained building blocks.
  • Templates define the view with binding, events, and logic.
  • Lifecycle hooks help manage setup and cleanup.
  • @Input() and @Output() manage data flow between components.
  • Two-way binding keeps data and UI in sync.
  • Use ViewChild for DOM or component access.
